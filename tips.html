Tips for using this tool:


1. Don't access nested elements directly. This because you can't do double subscripts in recipes.
Example:
    instead of: 
        grid[1][2] = 3
    use:
        grid[1] = row
        row[2] = 3

this also applies to nested dictionaries as well.




2. Please still check each line for mistakes. If you put that much trust in a
 random person's code to do your assignment, I'm very flattered, thank you,
 but please don't. Some more exotic aspects of python have not been implemented
 into the recipe formatter yet.

 Examples of recognized python features:
    - Lists
    - Dictionaries
    - For and while loops
    - Range
    - Random
    - methods
 Examples of unrecognized python features:
    - Sets
    - Floor division
    - List comprehensions
    - Default dictionaries
    - Type casting
    - Many many more




3. Why can't it write my inputs and outputs WAAAAAHH

The point of this tool is to show how stupid certain aspects of recipes are.
The fact a couple lines of replaceAll using regex can get you to 80% of a full
recipe is proof that writing recipe steps is redundant work, and is basically
porting your python program to another language. Inputs and outputs, however,
actually require brainpower to write, and to automate that task would require
something like ChatGPT. This aspect of recipies can at least be justified in 
the name of learning. This is not a tool for avoiding learning.
Sorry you'll still have to write your inputs and outputs yourself :3




4. Other stuff that may be useful
    - To italicize "var_name", add "var_name=" somewhere in your code text
    - To recognize "var_name" as a map, add "var_name={}" somewhere in your code text
    - To escape the stress of Comp140, click on the "about" button
